## 1. 전체 흐름

```
입력 이미지
    │
    ▼
[MediaPipe FaceLandmarker]
 478개 랜드마크 좌표 추출
    │
    ├─ 헤어라인 보정 (피부색 기반)
    └─ 눈썹 중심 계산 (좌우 랜드마크 평균)
    │
    ▼
핵심 좌표 확정
 hairline / brow_center / nose_tip / chin
 cheek_left / cheek_right
 forehead_left / forehead_right
 jaw_left / jaw_right
    │
    ▼
비율 계산
 R1 = 얼굴 길이 / 광대 너비
 R2 = 이마 너비 / 턱 너비
 R3 = 광대 너비 / (이마+턱)/2
 삼정 = 상안부 : 중안부 : 하안부
    │
    ▼
얼굴형 분류
 (학습된 고정 임계값 기반 if/else)
    │
    ▼
결과 출력
 달걀형 / 둥근형 / 각진형 / 하트형 / 긴형 / 역삼각형
```

## 2. 랜드마크 추출

### 사용 모델

Google MediaPipe **FaceLandmarker** (478개 랜드마크, 홍채 포함)

- 일반 얼굴 메시: 0 ~ 467번
- 좌측 홍채 중심: 468번
- 우측 홍채 중심: 473번

### 핵심 랜드마크 인덱스

| 포인트 | 인덱스 | 설명 |
|--------|--------|------|
| `hairline` | 10 + 피부색 보정 | 이마 끝 (헤어라인) |
| `brow_center` | 좌우 눈썹 평균 계산 | 상안부/중안부 경계 |
| `nose_tip` | 4 | 코끝 (중안부/하안부 경계) |
| `chin` | 152 | 턱끝 |
| `cheek_left` | 234 | 왼쪽 광대 끝 |
| `cheek_right` | 454 | 오른쪽 광대 끝 |
| `forehead_left` | 103 | 왼쪽 이마 |
| `forehead_right` | 332 | 오른쪽 이마 |
| `jaw_left` | 172 | 왼쪽 턱 |
| `jaw_right` | 397 | 오른쪽 턱 |

### 헤어라인 보정 (피부색 기반)

MediaPipe landmark 10은 이마 메시의 상단이지만, 실제 헤어라인과 일치하지 않을 수 있습니다.  
이를 보정하기 위해 **YCrCb 색공간에서 이마 피부 색상을 샘플링**한 뒤, landmark 10에서 위쪽으로 스캔하며 피부색이 끝나는 지점을 헤어라인으로 확정합니다.

```
landmark 10 (이마 확실한 피부)
    ↑ 위로 스캔
    ↑
    ↑ 피부색 유지 중 ...
    ★ 피부색 종료 → 헤어라인 확정
    │ (머리카락 또는 배경)
```

**핵심 로직:**
- landmark 10 아래 5~25px 구간의 피부 YCrCb 통계를 샘플링
- 피부 기준: 샘플 평균 ± 2.5 표준편차 (이미지별 적응형)
- 밝기(Y채널) 임계값 병행: 머리카락은 이마보다 50 이상 어두움
- 6행 연속 비-피부 감지 시 헤어라인 확정

### 눈썹 중심 계산

삼정 기준점으로 사용할 눈썹 위치를 좌우 눈썹 랜드마크 평균으로 계산합니다.

```python
left_brow_ids  = [70, 63, 105, 66, 107, 55, 65, 52, 53, 46]
right_brow_ids = [300, 293, 334, 296, 336, 285, 295, 282, 283, 276]
brow_y = 전체 20개 랜드마크 Y 평균
brow_x = 좌우 눈썹 중심 X 평균
```

## 3. 핵심 측정값 계산

모든 측정은 **픽셀 간 유클리드 거리** 또는 **Y좌표 차이**로 계산합니다.

```
         hairline  ●
              │
              │  ← 상안부 (hairline → brow_center)
              │
        brow_center  ●────────────────────── 이마 너비
              │
              │  ← 중안부 (brow_center → nose_tip)
              │
          nose_tip  ●
              │
              │  ← 하안부 (nose_tip → chin)
              │
            chin  ●

cheek_left ●──────────────────────────● cheek_right  ← 광대 너비
forehead_left ●──────────────────● forehead_right    ← 이마 너비
    jaw_left ●──────────────────────● jaw_right       ← 턱 너비
```

## 4. 삼정 비율

얼굴을 세로로 세 구간으로 나눠 균형을 측정합니다.

| 구간 | 범위 | 의미 |
|------|------|------|
| **상안부** | 헤어라인 → 눈썹 | 이마 영역 |
| **중안부** | 눈썹 → 코끝 | 눈·코 영역 |
| **하안부** | 코끝 → 턱끝 | 입·턱 영역 |

**표기 방식:** 가장 짧은 구간을 **1**로 기준 삼아 비율 표현

```
예시: 1.0 : 1.09 : 1.43  →  하안부가 가장 긴 얼굴
      1.0 : 1.0  : 1.0   →  이상적인 균형 (삼등분)
```

이상적인 황금 비율에 가까울수록 균형 잡힌 얼굴형으로 봅니다.

## 5. 얼굴형 분류 비율 R1·R2·R3

세 가지 비율로 얼굴의 가로 구조를 분석합니다.

### R1 — 얼굴 길이 비율

```
R1 = 얼굴 길이 (hairline → chin)
     ─────────────────────────────
        광대 너비 (cheek_left → cheek_right)
```

- **높음 (>1.5)**: 세로로 긴 얼굴
- **낮음 (<1.1)**: 가로로 넓은 얼굴

### R2 — 이마/턱 너비 비율

```
R2 = 이마 너비 (forehead_left → forehead_right)
     ─────────────────────────────────────────
        턱 너비 (jaw_left → jaw_right)
```

- **높음 (>1.1)**: 이마가 턱보다 넓음 → 하트형·역삼각형
- **낮음 (<0.9)**: 턱이 이마보다 넓음 → 각진형

### R3 — 광대 돌출 비율

```
R3 = 광대 너비 (cheek_left → cheek_right)
     ─────────────────────────────────────
     (이마 너비 + 턱 너비) / 2
```

- **높음 (>1.3)**: 광대가 이마·턱보다 발달 → 역삼각형·다이아몬드
- **낮음 (<1.1)**: 이마·턱·광대가 비슷한 너비 → 각진형·둥근형

### 얼굴형별 R1·R2·R3 경향

| 얼굴형 | R1 | R2 | R3 |
|--------|----|----|-----|
| 달걀형 | 중간 | 중간~높음 | 중간 |
| 둥근형 | 낮음 | 중간 | 낮음 |
| 각진형 | 중간 | 낮음 | 낮음 |
| 하트형 | 중간 | 높음 | 중간 |
| 긴형 | 높음 | 중간 | 중간 |
| 역삼각형 | 중간 | 높음 | 높음 |

## 6. 임계값 학습

### 왜 임계값을 학습하는가?

R1/R2/R3의 구체적인 경계값(예: "R1 > 1.43이면 긴형")은 얼굴 사진마다 달라지며 직관으로 정하기 어렵습니다.  
**라벨링된 데이터로 Decision Tree를 학습**하면 각 얼굴형을 가장 잘 구분하는 R1/R2/R3 임계값을 자동으로 찾아줍니다.

### 왜 Decision Tree인가?

| 모델 | 정확도 | 임계값 해석 | 런타임 속도 |
|------|--------|-------------|------------|
| Random Forest | 높음 | 어려움 (앙상블) | 느림 (300개 트리) |
| XGBoost | 높음 | 어려움 | 느림 |
| **Decision Tree** | 중~높음 | **직접 읽을 수 있음** | **즉각 (if/else)** |

Decision Tree의 각 **분기(split)가 곧 임계값**이므로, 학습 후 코드로 고정하면 런타임에 모델 로딩 없이 단순 비교 연산만으로 분류할 수 있습니다.

### 학습 데이터 구성

```
data/
├── 달걀형/    image001.jpg  image002.jpg  ...  (100장 내외)
├── 둥근형/    image001.jpg  ...
├── 각진형/    image001.jpg  ...
├── 하트형/    image001.jpg  ...
├── 긴형/      image001.jpg  ...
└── 역삼각형/  image001.jpg  ...
```

- 클래스당 **100장 내외** 권장 (적으면 임계값 불안정, 많을수록 정확)
- 다양한 조명·각도·인종을 포함할수록 일반화 성능 향상
- 정면 사진 위주 (측면·기울기가 심하면 랜드마크 오류)

### 학습 과정

```
이미지 폴더 입력
    │
    ▼
각 이미지에서 R1, R2, R3 자동 추출
 → face_shape_data.csv 저장
    │
    ▼
Decision Tree 학습
 (Random Forest와 교차 검증 정확도 비교)
    │
    ▼
Decision Tree 분기에서 임계값 추출
 예: R1 ≤ 1.3250 → ...
     R2 > 0.9810 → ...
    │
    ▼
shape_classification.py 에
 if/else 코드로 자동 고정
```

### 임계값 고정 예시 (학습 후 자동 생성)

```python
def _classify_trained(R1, R2, R3):
    if R1 <= 1.3250:
        if R2 <= 0.9810:
            if R3 <= 1.1200:
                return "각진형"
            else:
                return "둥근형"
        else:
            if R3 <= 1.3100:
                return "달걀형"
            else:
                return "역삼각형"
    else:  # R1 > 1.3250
        if R2 <= 1.1500:
            return "긴형"
        else:
            return "하트형"
```

학습 이후에는 이 함수만으로 분류가 완료되며, sklearn·모델 파일이 전혀 필요하지 않습니다.

## 7. 얼굴형 판단 규칙

학습 전 기본 규칙 (학습 후 자동 교체됩니다):

| 조건 | 얼굴형 |
|------|--------|
| R1 ≥ 1.50 | 긴형 |
| R1 ≤ 1.05 이고 R3 ≤ 1.15 | 둥근형 |
| R2 ≥ 1.15 이고 R3 ≤ 1.30 | 하트형 |
| R2 ≤ 0.92 이고 R3 ≤ 1.15 | 각진형 |
| R3 ≥ 1.35 | 역삼각형 |
| 위 조건 미해당 | 달걀형 (기본값) |

## 8. 실행 방법

### 단일 이미지 분석

```bash
python face_landmark_detection.py 입력이미지.jpg 결과이미지.jpg
```

출력 이미지에 표시되는 항목:
- R1, R2, R3 수치
- 삼정 비율 (상안부 : 중안부 : 하안부)
- 얼굴형 분류 결과

### 임계값 학습 (최초 1회)

```bash
# 이미지 폴더 방식 (권장)
python face_shape_train.py --images data/

# CSV 방식 (R1, R2, R3, face_shape 컬럼 필요)
python face_shape_train.py --csv face_shape_data.csv

# Decision Tree 깊이 조정 (기본 5, 깊을수록 정밀하지만 과적합 주의)
python face_shape_train.py --images data/ --depth 6
```

학습 완료 후 `shape_classification.py`가 자동 갱신되며, 이후 모든 분석에 고정 임계값이 적용됩니다.

### 분류 모듈 단독 테스트

```bash
python shape_classification.py
```

## 참고: 용어 정리

| 용어 | 설명 |
|------|------|
| 랜드마크 | 얼굴의 특정 위치를 나타내는 좌표 점 |

| 교차 검증 (CV) | 데이터를 여러 번 나눠 학습·평가를 반복해 모델 성능을 측정하는 방법 |
| 임계값 (threshold) | 두 클래스를 나누는 경계 수치 |
